# Workspace Adapter Sample

This sample shows the smallest possible adapter envelope.

## What to keep
- A clear adapter spec
- A small lifecycle
- Normalized output
- No core pollution

## What to extend later
- OS integration
- multi-agent coordination
- external project mapping
- richer capability reporting

## Why this sample exists
- To keep the adapter layer separate from `harness_core`
- To keep future integration work from becoming ad hoc
- To give the repository a concrete expansion point without thickening the core
