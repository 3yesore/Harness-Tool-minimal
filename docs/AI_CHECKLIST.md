# AI 维护检查清单

当 AI 接手维护 Harness 模块时，按此清单执行。

## 阶段 1：定位（30 秒）

- [ ] 读取项目主 INDEX.md
- [ ] 找到目标模块路径
- [ ] 确认模块存在

## 阶段 2：理解（1 分钟）

- [ ] 读取模块 INDEX.md
  - [ ] 理解模块职责（一句话）
  - [ ] 记录关键文件路径
  - [ ] 记录依赖关系
- [ ] 读取模块 SPEC.md
  - [ ] 理解输入格式和约束
  - [ ] 理解输出格式和错误码
  - [ ] 理解配置项
- [ ] **不要**此时读取全部代码

## 阶段 3：按需读码（按需）

- [ ] 根据任务只读取相关文件
- [ ] 参考 INDEX.md 的"关键文件"章节
- [ ] 如果需要理解依赖，读取依赖模块的 INDEX.md

## 阶段 4：执行变更（核心）

- [ ] 遵循 SPEC.md 的接口约束
- [ ] 如果修改接口，必须更新 SPEC.md
- [ ] 如果修改关键文件，必须更新 INDEX.md
- [ ] 保持代码风格一致

## 阶段 5：验证（必需）

- [ ] 运行冒烟测试：`python tests/smoke.py`
- [ ] 确保测试通过
- [ ] 如果测试失败，回滚或修复

## 阶段 6：记录（必需）

- [ ] 更新 CHANGELOG.md
  - [ ] 记录日期
  - [ ] 记录变更类型（Added/Changed/Fixed/Removed）
  - [ ] 简要描述变更
- [ ] 更新 INDEX.md 的"最后更新"
- [ ] 如果接口变更，在 CHANGELOG 标注"BREAKING CHANGE"

## 常见错误

❌ **错误 1**：一开始就读全部代码
- 正确做法：先读 INDEX + SPEC，按需读码

❌ **错误 2**：修改接口但不更新 SPEC
- 正确做法：接口变更必须同步更新 SPEC.md

❌ **错误 3**：不运行测试就提交
- 正确做法：每次变更后必须运行测试

❌ **错误 4**：不记录变更历史
- 正确做法：每次变更必须更新 CHANGELOG.md

## 快速参考

### 验证模块
```bash
python tools/validate_module.py <module_path>
```

### 运行测试
```bash
python <module_path>/tests/smoke.py
```

### 检查接口
```bash
cat <module_path>/SPEC.md
```

## 时间预算

- 定位：30 秒
- 理解：1 分钟
- 读码：2-5 分钟（按需）
- 变更：5-15 分钟
- 验证：1 分钟
- 记录：1 分钟

**总计**：通常 10-25 分钟完成一次维护任务
