# Harness Tool 版本规划

## 版本定位

### v1.0 - Minimal（当前仓库）
**目标用户**：个人开发者、小型项目

**核心功能**：
- ✅ 核心 Harness 规范（HARNESS_SPEC.md）
- ✅ 模块初始化工具（init_module.py）
- ✅ 合规性验证工具（validate_module.py）
- ✅ 半自动应用工具（apply_harness.py）
- ✅ 模板系统（INDEX/SPEC/CHANGELOG）
- ✅ Hello World 示例
- ✅ OpenClaw Skill 集成
- ✅ AI 维护检查清单

**特点**：
- 轻量级，零外部依赖
- 专注单模块维护
- 快速上手
- 总代码 < 1000 行，总大小 < 100KB

---

### v2.0 - Fetcher（规划中）
**目标用户**：Codex/Claude 用户、需要深度定制的开发者

**核心理念**：
为 Codex 和 Claude 等大模型专门优化，提供高度自定义能力。

**新增功能**：
- 🤖 **模型专属优化**
  - Codex 专用 prompt 模板
  - Claude 专用 prompt 模板
  - 模型特性适配（上下文窗口、推理模式）
- 🎨 **高自定义框架**
  - 自定义规范模板系统
  - 自定义验证规则引擎
  - 自定义 AI 工作流
- 📦 **智能模块理解**
  - 代码结构自动分析
  - 依赖关系自动提取
  - 接口自动推断（辅助生成 SPEC）
- 🔧 **增强工具**
  - `--fix` 自动修复常见问题
  - `--update` 更新已有文档
  - `--diff` 对比模块变化
- 📊 **项目级管理**
  - 多模块批量操作
  - 项目总索引自动生成
  - 模块依赖图可视化

**扩展规范**：
- 模型专属 prompt 规范
- 自定义规则 DSL
- 项目级 INDEX.md 规范

**使用场景**：
- 需要为特定 AI 模型优化的项目
- 需要深度定制规范的团队
- 管理 5-50 个模块的项目
- 需要智能辅助的代码重构

---

### v3.0 - Intruder（规划中）
**目标用户**：大型项目、企业级系统

**核心理念**：
深入企业开发流程，提供全方位治理和审计能力。

**新增功能**：
- 🚀 **CI/CD 深度集成**
  - GitHub Actions / GitLab CI 原生支持
  - PR 自动检查
  - 合规门禁
- 👥 **团队协作**
  - 代码审查清单自动生成
  - 交接文档自动生成
  - 责任矩阵追踪
- 📈 **审计追踪**
  - 变更历史完整追溯
  - 责任人记录
  - 合规性报告
- 🔐 **安全合规**
  - 敏感信息扫描
  - 权限验证
  - 安全策略执行
- 🌐 **多语言支持**
  - i18n 文档生成
  - 多语言规范模板
- 📊 **可视化仪表板**
  - 模块健康度监控
  - 依赖关系图
  - 治理进度追踪
- 🤖 **AI 辅助重构**
  - 自动识别需要 harness 化的代码
  - 智能建议重构方案
  - 风险评估
- 📦 **模块市场**
  - 共享/复用 harness 模块
  - 最佳实践库
  - 社区贡献

**扩展规范**：
- 企业级安全规范
- 合规性审计规范
- 团队协作规范
- SLA 定义规范

**使用场景**：
- 50+ 模块的大型系统
- 需要严格审计的项目
- 跨团队协作
- 企业级合规要求

## 迁移路径

### Minimal → Fetcher
1. 保持现有模块结构不变
2. 选择目标 AI 模型（Codex/Claude）
3. 应用模型专属 prompt 模板
4. 启用智能分析辅助
5. 配置自定义规则（可选）

### Fetcher → Intruder
1. 配置 CI/CD 流水线
2. 启用审计追踪
3. 集成团队协作工具
4. 部署可视化仪表板
5. 配置安全策略

## 版本选择指南

**选择 Minimal 如果**：
- 个人项目或小团队
- 模块数量 < 10
- 不需要 AI 模型专属优化
- 希望零依赖、快速上手

**选择 Fetcher 如果**：
- 深度使用 Codex 或 Claude
- 需要为特定 AI 优化工作流
- 需要高度自定义规范
- 模块数量 5-50
- 需要智能辅助重构

**选择 Intruder 如果**：
- 企业级项目
- 模块数量 > 50
- 需要严格审计和合规
- 跨团队协作
- 需要 CI/CD 深度集成
