# Harness 验证规则

## 必需文件

### INDEX.md
- 必须存在
- 必须包含以下章节：
  - `## 职责`
  - `## 关键文件`
  - `## 依赖`
  - `## 快速验证`
- 推荐包含：
  - `## 维护注意事项`
  - `## 最后更新`

### SPEC.md
- 必须存在
- 必须包含以下章节：
  - `## 输入`
  - `## 输出`
  - `## 配置`
  - `## 错误处理`
  - `## 示例`

## 推荐文件

### CHANGELOG.md
- 推荐存在
- 应记录所有重要变更
- 格式：日期 + 变更类型 + 描述

### tests/
- 必须存在目录
- 应包含至少一个可执行测试
- 测试应有清晰的成功/失败输出

### configs/
- 可选目录
- 如果存在，配置文件必须是有效的 JSON/YAML

## 验证级别

### 错误（Error）
- 缺少必需文件
- 必需章节缺失
- 配置文件格式错误
- 测试目录不存在

### 警告（Warning）
- 缺少推荐文件
- 推荐章节缺失
- 测试目录为空
- 配置目录不存在（如果模块需要配置）

## 严格模式

使用 `--strict` 标志时：
- 所有警告视为错误
- 验证失败会返回非零退出码
