# Harness 规范 v1

## 什么是 Harness

Harness 是一套让 AI 能够安全、高效接手模块维护的工程规范。

## 核心原则

### 1. 索引优先
- 每个模块必须有清晰的索引文件（INDEX.md）
- 索引说明模块职责、关键文件、依赖关系
- AI 通过索引快速定位，而非全量扫描

### 2. 规范化接口
- 输入输出必须标准化
- 配置与代码分离
- 错误处理统一

### 3. 最小上下文
- 关键信息集中在索引与规范
- 代码按需阅读
- 避免冗余文档

### 4. 可验证
- 每个模块必须有验证方式
- 支持快速冒烟测试
- 明确的成功/失败标准

### 5. 可交接
- 新 AI 能通过索引 + 规范快速上手
- 支持跨会话、跨 AI 的迭代
- 变更历史可追溯

## 模块结构

```
module_name/
├── INDEX.md           # 模块索引（必需）
├── SPEC.md            # 接口规范（必需）
├── CHANGELOG.md       # 变更历史（推荐）
├── src/               # 源代码
├── tests/             # 测试
├── configs/           # 配置
└── docs/              # 详细文档（可选）
```

## INDEX.md 模板

```markdown
# Module: [模块名]

## 职责
[一句话说明模块做什么]

## 关键文件
- `src/main.py`: 入口
- `configs/default.json`: 默认配置
- `tests/smoke.py`: 冒烟测试

## 依赖
- 外部依赖: [列出]
- 内部依赖: [列出其他模块]

## 快速验证
```bash
python tests/smoke.py
```

## 维护注意事项
[列出关键约束、已知问题等]
```

## SPEC.md 模板

```markdown
# 接口规范: [模块名]

## 输入
[定义输入格式、参数、约束]

## 输出
[定义输出格式、字段、错误码]

## 配置
[定义配置项、默认值、环境变量]

## 错误处理
[定义错误类型、处理策略]

## 示例
[提供完整的调用示例]
```

## AI 工作流

1. **定位模块**：读取主索引，找到目标模块
2. **理解接口**：读取 INDEX.md + SPEC.md
3. **按需读码**：根据任务读取关键代码
4. **执行变更**：遵循规范进行修改
5. **验证结果**：运行测试，更新文档
6. **记录变更**：更新 CHANGELOG.md

## 规范演进

- 本规范版本：v1
- 变更提案：通过 PR 提交到主仓库
- 向后兼容：新版本必须兼容旧模块
